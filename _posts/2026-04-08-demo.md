The MARS validation campaign uses six demonstrations, each designed to prove a specific claim about the coordination architecture. This post maps each demonstration to the claim it validates and explains the design choice behind it.

---

## Demo 1: Pick and Place (Asynchronous Mode)

**Claim**: dual-arm asynchronous operation achieves measurable throughput gains over sequential single-arm operation.

**Design**: both arms execute pick-and-place cycles simultaneously on spatially partitioned targets. A single-arm baseline (same task, same hardware, sequential) provides the comparison.

**What it proves**: the coordination architecture enables genuine parallel operation with no inter-arm coupling. The 28.6% cycle time reduction is observable and reproducible (0.5% CV across trials).

**What it does not prove**: that asynchronous mode is safe in shared workspace. Safety relies on spatial partitioning — the targets are in separate zones. This is a throughput demo, not a collision safety demo.

## Demo 2: Hand Over (Hybrid Mode)

**Claim**: hybrid coordination enables safe object handoff in the shared workspace centre region.

**Design**: arm_1 approaches the centre zone carrying an object; arm_2 approaches from the opposite side to receive it. Planning uses the `dual` 12-DOF group (full collision awareness); execution is dispatched to independent controllers.

**What it proves**: inter-arm collision checking at planning time allows safe approach to handover position without physical contact. The planner correctly identifies valid configurations for both arms within the shared zone.

**What it does not prove**: the gripper mechanics of object transfer (MARS does not model grasp forces) or timing tolerance of the handover (both arms approach simultaneously but not in true lockstep).

## Demo 3: Cross Over (Hybrid/Synchronous Mode)

**Claim**: conservative collision padding does not prevent valid workspace maneuvers — arms can navigate shared space with appropriate planning.

**Design**: arm_1 is commanded to a target on arm_2's side of the workspace, requiring its trajectory to cross through the shared centre zone while arm_2 holds a static position.

**What it proves**: the planner finds collision-free paths through the shared workspace without requiring complete workspace partitioning. The 85% planning success rate (across 5+ crossing attempts) shows these paths are achievable, and 0/5 executions produce physical collision.

**What it does not prove**: that arbitrary crossing maneuvers are always achievable. The 15% planning failures represent configurations where the padded volumes cannot be navigated without intersection.

## Demo 4: Async Flexibility (True Asynchronous Motion)

**Claim**: asynchronous mode allows each arm to complete genuinely different pose sequences at its own pace.

**Design**: arm_1 executes 3 poses; arm_2 executes 2 poses. Both run in parallel. arm_1 finishes later than arm_2 — there is no shared timing.

**What it proves**: the asynchronous execution architecture imposes no artificial synchronisation. Arms complete at different times when their task sequences differ in length. The temporal independence is visible in the motion profiles.

**What it does not prove**: independence at the statistical level (per-joint correlations reflect shared mechanical state, not coupled control).

---

## Demo 5: Cooperative Manipulation (Synchronous Mode)

**Claim**: synchronous mode maintains stable relative positioning between end-effectors throughout motion.

**Design**: both arms grasp a single object at two contact points and move it from pose A to pose B. Relative drift between grasp points is tracked throughout motion.

**What it proves**: 12-DOF planning constrains both arms to move such that their relative configuration is maintained. The planner generates trajectories where both arms arrive at each waypoint simultaneously (within the dispatch offset).

**What it does not prove**: force distribution or grasp stability (no force sensing on the NED2). Whether the object actually moved as commanded depends on grip security, which is not instrumented.
